TraceInv.ComputeTraceOfInverse.CholeskyMethod module¶

TraceInv.ComputeTraceOfInverse.CholeskyMethod.CholeskyMethod(A, UseInverseMatrix=True)[source]¶

Computes trace of inverse of matrix using Cholesky factorization by

\[\mathrm{trace}(\mathbf{A}^{-1}) = \| \mathbf{L}^{-1} \|_F^2\]

where \(\mathbf{L}\) is the Cholesky factorization of \(\mathbf{A}\), and \(\| \cdot \|_F\) is the Frobenius norm.

Note

This function does not produce correct results when 'A' is sparse. It seems sksparse.cholmod has a problem.

When A = K, it produces correct result, But when A = Kn = K + eta I, its result is differen than Hurchinson, Lanczos method. Also its result becomes correct when A is converted to dense matrix, and when we do not use skspase.cholmod anymore.

Parameters

  • A (ndarray) – Invertible matrix

  • UseInverseMatrix (bool) – Flag to invert Cholesky matrix. If false, the inverse of Cholesky is not computed, but a linear system is solved.

Returns

Trace of matrix A.

Return type

float

TraceInv.ComputeTraceOfInverse.CholeskyMethod.ComputeTraceInvByInverseOfCholesky(L, UseSparse)[source]¶

Compute the trace of inverse by inverting the Cholesky matrix L directly.

Note

  • For large matrices: This method is very slow and results are unstable.

  • UseSparse (bool) – Flag, if true, the matrix L is considered as sparse.

TraceInv.ComputeTraceOfInverse.CholeskyMethod.ComputeTraceInvBySolvingLinearSystem(L, n, UseSparse)[source]¶

Computes the trace of inverse by solving a linear system for Cholesky matrix and each column of the identity matrix to obtain the inverse of L sub-sequentially.

The matrix \(\mathbf{L}\) is not inverted directly, rather, the linear system

\[\mathbf{L} \boldsymbol{x}_i = \boldsymbol{e}_i, \qquad i = 1,\dots,n\]

is solved, where \(\boldsymbol{e}_i = (0,\dots,0,1,0,\dots,0)^{\intercal}\) is a column vector of zeros except its \(i\) th entry is one, and \(n\) is the size of the square matrix \(\mathbf{A}\). The solution \(\boldsymbol{x}_i\) is the \(i\) th column of \(\mathbf{L}^{-1}\). Then, its Frobenius norm is

\[\| \mathbf{L} \|_F^2 = \sum_{i=1}^n \| \boldsymbol{x}_i \|^2.\]

The method is memory efficient as the vectors \(\boldsymbol{x}_i\) do not need to be stored, rather, their norm can be stored in each iteration.

Note

This method is slow, and it should be used only if the direct matrix inversion can not be computed (such as for large matrices).

Parameters

  • L (ndarray) – Cholesky matrix

  • UseSparse (bool) – Flag, if true, the matrix L is considered as sparse.

Returns

Trace of matrix A.

Return type

float

TraceInv.ComputeTraceOfInverse.ComputeTraceOfInverse module¶

TraceInv.ComputeTraceOfInverse.ComputeTraceOfInverse.ComputeTraceOfInverse(A, ComputeMethod='cholesky', **Options)[source]¶

Computes the trace of inverse of a matrix. See Compute Trace of Inverse User Guide for details.

Parameters

  • A (ndarray) – Invertible matrix

  • ComputeMethod (string) – One of 'cholesky', 'hutchinson', or 'SLQ'. Default if 'cholesky'.

  • Options (**kwargs) – Options for either of the methods.

Returns

trace of inverse of matrix

Return type

float

Raises

RunTimeError – Method is not recognized.

Methods:

The trace of inverse is computed with one of these three methods in this function.

'ComputeMethod'

Description

Type

'cholesky'

Cholesky method

exact

'hutchinson'

Hutchinson method

approximation

'SLQ'

Stochastic Lanczos Quadrature

approximation

Depending the method, this function calls these modules:

  • TraceInv.ComputeTraceOfInverse.CholeskyMethod

  • TraceInv.ComputeTraceOfInverse.HutchinsonMethod

  • TraceInv.ComputeTraceOfInverse.StochasticLanczosQuadratureMethod

Examples:

>>> from TraceInv import GenerateMatrix
>>> from TraceInv import ComputeTraceOfInverse

>>> # Generate a symmetric positive-definite matrix of the shape (20**2,20**2)
>>> A = GenerateMatrix(NumPoints=20)

>>> # Compute trace of inverse
>>> trace = ComputeTraceOfInverse(A)

The above example uses the Cholesky method by default. In the next example, we apply the Hutchinson’s randomized estimator method.

>>> trace = ComputeTraceOfInverse(A,ComputeMethod='hutchinson',NumIterations=20)

Using the stochastic Lanczos quadrature method with Lanczos tri-diagonalization

>>> trace = ComputeTraceOfInverse(A,ComputeMethod='SLQ',NumIterations=20,LanczosDegree=30)

Using the stochastic Lanczos quadrature method with Golub-Kahn bi-diagonalization

>>> trace = ComputeTraceOfInverse(A,ComputeMethod='SLQ',NumIterations=20,LanczosDegree=30,UseLanczosTridiagonalization=False)

TraceInv.ComputeTraceOfInverse.HutchinsonMethod module¶

TraceInv.ComputeTraceOfInverse.HutchinsonMethod.HutchinsonMethod(A, NumIterations=20)[source]¶

Computes the trace of inverse of a matrix by Hutchinson method.

The random vectors have Rademacher distribution. Compared to the Gaussian distribution, the Rademacher distribution yields estimation of trace with lower variance.

Note

In the is function, the generated set of random vectors are orthonormalized using QR decomposition. Hence, they no longer have Rademacher distribution. By orthonormalization, the solution seem to have better convergence.

Parameters

  • A (ndarray) – invertible matrix

  • NumIterations (int) – number of Monte-Carlo random trials

Returns

Trace of matrix A.

Return type

float

TraceInv.ComputeTraceOfInverse.StochasticLanczosQuadratureMethod module¶

TraceInv.ComputeTraceOfInverse.StochasticLanczosQuadratureMethod.StochasticLanczosQuadratureMethod(A, NumIterations=20, LanczosDegree=20, UseLanczosTridiagonalization=False)[source]¶

Computes the trace of inverse of matrix based on stochastic Lanczos quadrature method.

Reference:

  • Ubaru, S., Chen, J., and Saad, Y. (2017), Fast Estimation of \(\mathrm{tr}(F(A))\) Via Stochastic Lanczos Quadrature, SIAM J. Matrix Anal. Appl., 38(4), 1075-1099.

Note

In Lanczos tri-diagonalization method, \(\theta\) is the eigenvalue of T. However, in Golub-Kahn bi-diagonalization method, \(\theta\) is the singular values of B. The relation between these two methods are are follows: B.T*B is the T for A.T*A. That is, if we have the input matrix A.T*T, its Lanczos tri-diagonalization T is the same matrix as if we bi-diagonalize A (not A.T*A) with Golub-Kahn to get B, then T = B.T*B. This has not been highlighted paper in the above paper.

To correctly implement Golub-Kahn, here \(\theta\) should be the singular values of B, NOT the square of the singular values of B (as described in the above paper incorrectly!).

Parameters

  • A (ndarray) – invertible matrix

  • NumIterations (int) – Number of Monte-Carlo trials

  • LanczosDegree (int) – Lanczos degree

  • UseLanczosTridiagonalization (bool) – Flag, if True, it uses the Lanczos tri-diagonalization. If False, it uses the Golub-Kahn bi-diagonalization.

Returns

Trace of A

Return type

float
