TraceInv.ComputeTraceOfInverse.CholeskyMethod module¶

TraceInv.ComputeTraceOfInverse.CholeskyMethod.CholeskyMethod(A, UseInverseMatrix=True)[source]¶

Computes trace of inverse of matrix using Cholesky factorization.

\[\mathrm{trace}(\mathbf{A}^{-1}) = \| \mathbf{L}^{-1} \|_F^2\]

where \(\mathbf{L}\) is the Cholesky factorization of \(\mathbf{A}\), and \(\| \cdot \|_F\) is the Frobenius norm.

Note

This function does not produce correct results when 'A' is sparse. It seems sksparse.cholmod has a problem.

When A = K, it produces correct result, But when A = Kn = K + eta I, its result is differen than Hurchinson, Lanczos method. Also its result becomes correct when A is converted to dense matrix, and when we do not use skspase.cholmod anymore.

Parameters

  • A (ndarray) – Invertible matrix

  • UseInverseMatrix (bool) – Flag to invert Cholesky matrix. If false, the inverse of Cholesky is not computed, but a linear system is solved.

Returns

Trace of matrix A.

Return type

float

TraceInv.ComputeTraceOfInverse.LinearAlgebra module¶

TraceInv.ComputeTraceOfInverse.LinearAlgebra.GolubKahnLanczosBidiagonalization(A, w, m, Tolerance=1e-08)[source]¶
Inputs:

A: symmetric matrix, n*n w: a random vector, n*1 m: Lanczos degree of bidoagonalization, scalar.

Output:

B: symmetric matrix, upper bidiagonal, (m+1)*(m+1)

Note

  • A should be symmetric and positive-definite matrix size n*n.

  • B will be symmetric and positve-definite matrix of size (m+1,m+1).

Bidiagonalizes matrix A to B using the start vector w. m is the Lanczos degree, which will be the size of square matrix B.

Reference

  • NetLib Algorithm 6.27

  • Matrix Computations, Golub, p. 495

  • Templates for Solution of Algebraic Eigenvalue Problem, Demmel, p. 143

Issues:

When matrix A is very close to the identity matrix, the Golub-Kahn bidoagonalization method can not find \(\beta\), as \(\beta\) becomes zero. If A is not exactly identity, you may descrease the Tolerance to a very small number. However, if A is almost identity matrix, descreasing tolerance will not help, and this function cannot be used.

Parameters

  • A (ndarray) – Input matrix of the size n*n. Matrix should be positive-definite and symmetric.

  • w (array) – Start vector for the Lanczos tri-diagonalization. Column vector of size n. It could be generated randomly. Often it is generated by the Rademacher distribution with entries +1 and -1.

  • m (int) – Lanczos degree, which is the number of Lanczos iterations.

  • Tolerance (float) – The tolerance of the residial error of the Lanczos iteration.

Returns

Symmetric positive-definite matric B of the size (m+1)*(m+1).

Return type

ndarray

TraceInv.ComputeTraceOfInverse.StochasticLanczosQuadratureMethod module¶

TraceInv.ComputeTraceOfInverse.StochasticLanczosQuadratureMethod.StochasticLanczosQuadratureMethod(A, NumIterations=20, LanczosDegree=20, UseLanczosTridiagonalization=False)[source]¶

Computes the trace of inverse of matrix based on stochastic Lanczos quadrature method.

Reference

  • Ubaru, S., Chen, J., and Saad, Y. (2017), Fast Estimation of \(\mathrm{tr}(F(A))\) Via Stochastic Lanczos Quadrature, SIAM J. Matrix Anal. Appl., 38(4), 1075-1099.

Note

In Lanczos tridiagonalization method, \(\theta\) is the eigenvalue of T. However, in Golub-Kahn bidoagonalization method, \(\theta\) is the singular values of B. The relation between these two methods are are follows: B.T*B is the T for A.T*A. That is, if we have the input matrix A.T*T, its Lanczos tridiagonalization T is the same matrix as if we bidiagonalize A (not A.T*A) with Golub-Kahn to get B, then T = B.T*B. This has not been highlighted paper in the above paper.

To correctly implement Golub-Kahn, here \(\theta\) should be the singular values of B, NOT the square of the singular values of B (as decribed in the above paper incorrectly!).

Parameters

  • A (ndarray) – invertible matrix

  • NumIterations (int) – Number of Monte-Carlo trials

  • LanczosDegree (int) – Lanczos degree

  • UseLanczosTridiagonalization (bool) – Flag, if True, it uses the Lanczos tridiagonalization. If False, it uses the Golub-Kahn bi-diagonalization.

Returns

Trace of A

Return type

float
