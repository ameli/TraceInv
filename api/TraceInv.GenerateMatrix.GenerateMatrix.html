GenerateMatrix¶

TraceInv.GenerateMatrix.GenerateMatrix(NumPoints=20, DecorrelationScale=0.1, nu=0.5, UseSparse=False, GridOfPoints=True, KernelThreshold=0.03, RunInParallel=False, Plot=False, Verbose=False)[source]¶

Generates symmetric and positive-definite matrix for test purposes.

The generated matrix is a correlation matrix based on Matern correlation of spatial distance of a list of points in the unit square. The Matern correlation function can be characterized by two parameters:

  • \(\rho\): the discorrelation scale in \(\rho \in (0,1]\). Smaller decorrelation produces correlation matrix that is closer to the identity matrix.

  • \(\nu\): correlation smoothness, \(\rho > 0\). Larger smoothness produces correlation similar to the Gaussian process. Smaller smoothness produces more discrete correlation function.

Also, the size of the generated matrix is determined by NumPoints which we refer to as \(n\), and GridOfPoints.

  • If GridOfPoints is True (default value), then, the matrix is of the size \(n^2 \times n^2\).

  • If GridOfPoints is False, then, the matrix is of the size \(n \times n\).

The values of the correlation matrix are between \(0\) and \(1\). To sparsify the generated matrix, set KernelThreshold, which makes all correlations below this threshold to be zero. If KernelThreshold is set to zero, the matrix is not sparsified.

Note

Too large KernelThreshold might eradicate the positive-definiteness of the correlation matrix.

For very large sparse matrices, use RunInParallel to generate the rows and columns of the correlation matrix in parallel.

Note

To use parallel processing, the package ray should be installed.

Parameters

  • NumPoints (int) – Depending on GridOfPoints, this is either the number of points or the number of grid points along an axis.

  • DecorrelationScale (float) – A parameter of correlation function that scales distance.

  • nu (float) – A parameter of the Matern correlation function. nu modulates the smoothness of the stochastic process.

  • UseSparse (bool) – Flag to indicate the correlation matrix should be sparse or dense matrix.

  • KernelThreshold – To sparsify the matrix (if UseSparse is True), the correlation function values below this threshold is set to zero.

  • RunInParallel (bool) – Runs the code in parallel. Note that the ray module should be uncommented.

  • Plot (bool) – If True, the matrix will be plotted.

  • Verbose (bool) – If True, prints some information during the process. Default is False.

Returns

Correlation matrix. If x and y are n*1 arrays, the correlation K is n*n matrix.

Return type

numpy.ndarray or sparse array

Example:

Generate a matrix of the shape (20**2,20**2) based on mutual correlation of a grid of 20x20 points on unit square

>>> from TraceInv import GenerateMatrix
>>> A = GenerateMatrix(NumPoints=20)

Generate a correlation matrix of shape (20,20) based on 20 random points in unit square. Default for GridOfPoints is True.

>>> A = GenerateMatrix(NumPoints=20,GridOfPoints=False)

Generate a matrix of shape (20**2,20**2) with stronger spatial correlation. Default for DecorrelationScale is 0.1.

>>> A = GenerateMatrix(NumPoints=20,DecorrelationScale=0.3)

Generate a correlation matrix with more smoothness.. Default for nu is 0.5.

>>> A = GenerateMatrix(NumPoints=20,nu=2.5)

Sparsify correlation matrix (makes all entries below 0.03 to zero). Default for KernelThreshold is 0.03.

>>> A = GenerateMatrix(NumPoints=20,UseSparse=True,KernelThreshold=0.03)

For very large correlation matrices, generate the rows and columns are generated in parallel. To use RunInParallel option, the package ray should be installed.

>>> A = GenerateMatrix(NumPoints=100,UseSparse=True,RunInParallel=True)

Plot the matrix by

>>> A = GenerateMatrix(NumPoints=30,Plot=True)
